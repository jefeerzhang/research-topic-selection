# 案例：轻量启动（只有题目和摘要）

这个案例展示用户没有完整文献笔记，只有初步想法和一段摘要时如何启动。

## 用户输入

- **主题**：数字游民的工作-生活边界研究
- **已有材料**：仅一段自己写的摘要

```text
我想研究数字游民如何在远程工作模式下管理工作与生活边界。
初步问题是：灵活办公是否模糊了传统工作-生活边界，并带来新的倦怠形式？
已有材料：一篇我自己写的 500 字摘要。
```

## 启动 prompt

```text
我想用 research-topic-selection 来评估一个选题。我的主题是：
“数字游民的工作-生活边界研究”。

我已有的材料：
- 一段 500 字的摘要（见下）

摘要：我想研究数字游民如何在远程工作模式下管理工作与生活边界...

请用文献综述作为交付物，学科分支为质性案例，时间窗为 2020-2026，语言为中文。
```

## 预期行为

- Agent 会先把这段摘要保存为 `user_materials/` 下的 Markdown 文件；
- 运行 `register_material.py` 登记材料；
- 由于材料较薄，进入扫描后需要补充至少 2-3 篇核心文献或数据线索；
- 最终产物可能是一个更聚焦的研究问题，例如：
  - “数字游民的时间边界策略如何影响其倦怠水平？”
- 会明确标注：基于薄材料，最终选题置信度为“待验证”，需要两周 pilot。

## 关键审查点

- 500 字摘要是否足以支撑一个学位论文选题？
- 数字游民样本是否可接触？
- 工作-生活边界与“传统倦怠”量表是否适用？
- 如果无法获取数字游民样本，是否应暂缓？
