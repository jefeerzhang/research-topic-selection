# 文献矩阵生成规范（Phase 3.5）

> **加载时机**：Phase 3 问题域地图之后、Phase 4 文献脉络之前。
> **目的**：把已有文献放入"研究问题 × 方法"二维表，可视化领域结构和潜在研究空白。

## 一、为什么需要矩阵

问题域地图（Phase 3）回答"这个领域由什么构成"，是叙述性的、可比较性弱。文献矩阵把文献放入二维坐标，让"哪些问题已被哪些方法做过、哪些组合没人做"一眼可见。

矩阵不是替代判断的工具，而是生成**候选研究缺口**的工具：

- 空白格不必然等于有价值的缺口；
- 但空白格比"凭感觉觉得没人做"有依据；
- 进入 Phase 6 核心缺口判断时，矩阵空白格是输入之一，仍需经过"已解释/仍不足/为何重要"三段式分析。

## 二、矩阵结构

二维表：

- **Y 轴（行）**：研究问题 / 现象 / 应用场景
  - 从 `03_五维扫描.md` 的"学术文献扫描"维度提取主要分支；
  - 从 `04_问题域地图.md` 的"主要学术分支"和"潜在研究切口"提取；
  - 也可来自 `02A` 用户材料研判中识别的主题。
- **X 轴（列）**：方法 / 数据 / 理论视角
  - 从已有论文的"Core Mechanism"或方法描述归类；
  - 常见类别：面板计量、案例研究、田野调查、文本分析、实验设计、理论建模、二手数据、混合方法。
- **单元格**：
  - 填入对应的论文 ID 或 `material_id`；
  - 空白格默认标注"待验证"，不直接判定为缺口。

## 三、生成步骤

1. **收集候选轴标签**
   - 从 `03_五维扫描.md`、`04_问题域地图.md` 和 `02_用户材料研判.md` 提取候选研究问题；
   - 从用户打包文献的全文（`user_materials/extracted/*.md`）提取候选方法；
   - 不预设数量，通常 3-7 个 Y 轴标签、3-6 个 X 轴标签较为合适。

2. **归一化**
   - 语义相同但表述不同的标签合并（如"面板数据"和"panel data"合并为"面板数据"）；
   - 归一化当前为规则匹配（基于关键词），LLM 辅助归一化待后续版本；
   - 归一化结果在矩阵文件的"归一化说明"段说明，避免误判。

3. **填充单元格**
   - 对每篇提取的文献，根据其方法和研究问题归入对应单元格；
   - 一篇文献可以归入多个单元格（如果跨方法或跨问题）；
   - 单元格中的论文按文献 ID 列出，方便溯源。

4. **标注空白格**
   - 所有空白单元格默认为"待验证"，**不直接判定为缺口**；
   - 在矩阵文件的"空白格说明"段明确：哪些空白格可能是真正缺口，哪些可能是检索不全或归一化失败。

5. **导出**
   - `03A_文献矩阵.md`：人类可读的 Markdown 表格；
   - `review/matrix.json`：结构化数据，供 Phase 4 文献脉络和 Phase 6 核心缺口引用。

## 四、产物格式

### 4.1 `03A_文献矩阵.md`

```markdown
# 文献矩阵（Phase 3.5）

> 基于 Phase 1.7 提取的论文全文 + Phase 2 学术文献扫描 + Phase 3 问题域地图
> Y 轴：研究问题 / 应用场景
> X 轴：方法 / 数据 / 理论视角

## 矩阵

| 研究问题 \ 方法 | 面板计量 | 案例研究 | 田野调查 | 文本分析 |
|---|:---:|:---:|:---:|:---:|
| 算法评分与离职率 | M-aa11, M-bb22 | | | |
| 平台规则透明度 | | M-cc33 | | M-dd44 |
| 收入波动机制 | M-ee55 | | M-ff66 | |
| 职业安全感 | | | | |

## 归一化说明

- "面板数据" 和 "panel data" 合并为 "面板计量"；
- "deep interview" 归入 "田野调查"。

## 空白格说明（待验证）

| 空白组合 | 可能的解释 | 是否进入核心缺口候选 |
|---|---|---|
| 职业安全感 × 面板计量 | 无定量研究或归一化失败 | 是，需在 Phase 6 验证 |
| 算法评分与离职率 × 案例研究 | 可能缺乏典型案例 | 待定 |
```

## 六、闸门校验

`selection_gate.py --enter scan-review` 在原有基础上增加矩阵校验：

1. `03A_文献矩阵.md` 文件存在且 ≥300 字符；
2. 包含 Markdown 表格（`| ... | ... |` 至少 3 行）；
3. 包含"空白格说明"段；
4. `review/matrix.json` 是合法 JSON，`y_axis`、`x_axis`、`cells` 均为数组；
5. `empty_cells` 中至少标注 `candidate_gap: true` 的项，否则视为"无候选缺口"，需在 `04_问题域地图.md` 中说明。

## 七、局限与边界

- 矩阵归一化仍是规则匹配，标签语义相近但表述差异大时可能漏合并；
- 空白格可能是"该组合确实无人做"、"检索不全"、"归一化失败"中的任何一种，不能直接当缺口；
- 矩阵仅基于用户提供的文献，独立检索发现的新论文（如有）应同时纳入；
- 矩阵是辅助判断工具，不是替代判断。Phase 6 仍需结合趋势、扫描发现和材料冲突做最终筛选。
