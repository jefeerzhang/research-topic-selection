# 好问题闸门

> 加载时机：完成 `07_核心缺口.md` 后、生成 `08_选题推荐.md` 前。

本闸门把“有缺口的方向”转化为“重要、可检验、可证伪且能尽快试错的研究问题”。它不重复五维扫描，也不以“尚无人研究”代替研究价值。

方法参考并适配自 [Rimagination/good-question](https://github.com/Rimagination/good-question)（MIT License）。本文件已针对经管、社科论文和课题申报流程压缩重组，运行时不依赖外部技能。

## 一、证据边界

只使用 `03_五维扫描.md`、`04_问题域地图.md`、`05_文献脉络.md`、`06_趋势判断.md`、`07_核心缺口.md` 和用户材料中已有的事实。新增时效性或学科事实必须先检索。

将关键判断标记为：

- 来源支持：资料直接支持。
- 综合推断：由多条证据推导，但来源未直接表述。
- 尚待验证：当前证据不足，列入下一步核验。

不得把“没有检索到”写成“尚无人研究”。

## 二、发散候选问题

围绕 1-3 个核心缺口生成 5-10 个候选问题。混合使用以下透镜，避免只换标题不换问题：

- 重要性与可行性：谁会因答案改变判断或行动？现有资源能否提供可信证据？
- 默认假设挑战：现有研究把什么视为固定、显然或无需解释？
- 竞争性解释：同一现象还可能由哪些机制产生？
- 边界条件：主流理论或方法在何种对象、时期、制度或情境下失效？
- 条件变化：哪些旧结论因政策、技术、数据或行为环境变化值得重检？
- 简化检验：复杂模型相较简单基线究竟增加了什么解释力？
- 利益相关者转换：研究结论对不同主体是否方向相反？

在 `07A_好问题卡.md` 中使用 `### 候选问题 1` 至 `### 候选问题 10` 标题。每个候选注明唯一的
`候选 ID`（如 Q1），并写明核心问句、对应缺口、挑战的假设或张力。相同 ID 和评分同步写入
`review/question_scores.json`，格式见 `references/artifact-schemas.md`。

## 三、评分与淘汰

对每个候选按 1-5 分评分，并在“候选问题评分”表中保留理由：

| 指标 | 判断标准 |
|---|---|
| 重要性 | 对理论、实践、政策或方法是否有实质后果 |
| 可行性 | 现有时间、数据、材料和能力能否形成可信证据 |
| 可证伪性 | 能否指出会削弱或推翻主张的观察结果 |
| 证据杠杆 | 一个小规模 pilot 能否显著改变继续或放弃的判断 |
| 原创性 | 是否挑战重要假设或形成非平凡的新组合 |
| 负向结果价值 | 结果不支持预期时是否仍产生可报告的知识 |

出现任一情况即淘汰或转为备选：

- 只有“没人做过”，说不清为什么重要。
- 无法指出受益者、理论赌注或现实后果。
- 无法提出合理的竞争性解释或推翻条件。
- 先确定方法，再寻找一个可以套用的问题。
- 所需资源明显超过用户约束。
- 增加复杂性，却说不清复杂性换来了什么。

## 四、Grill 式压力测试（v1.5.2 新增）

对评分排名最高的 1-3 个候选问题，主线程采用 `grill-me` 思路进行逐字段追问，一次只解决一个决策分支，
每个问题附带推荐答案，直到用户确认或纠正。

### 4.1 追问顺序

按以下 **6 个关键字段**逐项追问并完善好问题卡：

1. **核心研究问题**：这个候选问句是否准确捕捉了你想回答的问题？
2. **为什么值得做**：谁会因答案改变判断？理论、实践或政策上的后果是什么？
3. **默认假设与竞争性解释**：现有研究把什么视为固定或显然？同一现象还可能由哪些机制产生？
4. **关键判别证据与推翻条件**：什么样的观察或 pilot 能区分这些竞争性解释？什么结果会让这个研究问题不再成立？
5. **两周 pilot 与下一步动作**：两周内能做什么来支持继续、修改或停止？确认后第一个要执行的具体动作是什么？
6. **资源需求与最强评审质疑**：需要哪些数据、田野、合作方或伦理审批？评审专家最可能攻击的点是什么？

### 4.2 追问原则

- **一次只问一个字段**：不一次性抛出整张卡片让用户填写。
- **每个问题附带推荐答案**：例如"基于你的材料和扫描，我建议两周 pilot 做 X，因为 Y。你同意吗？"
- **用户纠正即更新**：用户修改后，立即写入 `07A_好问题卡.md` 相应字段，不沿用旧值。
- **不强迫共识**：如果用户对某个字段坚持不同意见，记录分歧并标注风险，作为后续选题判断的输入。

### 4.3 终止条件

当 1-3 张好问题卡的所有必填字段都经过用户确认或明确标注分歧，且满足本文件"形成好问题卡"的最低成熟度标准时，
停止追问，进入"映射到 3+2 选题"。

## 五、形成好问题卡

选择排名最高的 1-3 个问题。每张卡使用 `## 好问题卡 1` 形式的标题，并完整填写：

```markdown
## 好问题卡 1

**卡片 ID：** Q1
**暂定题目：**
**核心研究问题：**
**为什么值得做：**
**挑战了什么默认假设：**
**竞争性解释：** H1；H2；必要时 H3
**关键判别证据或实验：**
**什么结果会推翻它：**
**两周内可做的 pilot：**
**需要的数据/资源：**
**最强评审质疑：**
**下一步动作：**
```

卡片不成熟时不得进入 3+2 推荐。至少满足：研究意义明确；存在两个合理解释；判别证据可观察；推翻条件不为空；pilot 能在两周左右形成继续、修改或停止的决策。

## 六、映射到 3+2 选题

- 三个主推选题必须在“来源好问题卡”填写结构化卡片 ID，并保留卡片的核心研究问题。
- 同一张卡可以派生不同对象、识别策略或情境边界，但不得伪装成三个独立问题。
- 备选选题优先来自被保留的低风险变体，并写明何时从主推项降级到该备选。
- 方法、数据和题目措辞可以在 Phase 7 调整，但若改变核心问题、竞争性解释或推翻条件，必须同步更新 `07A_好问题卡.md`，旧 topics 审查随 hash 变化失效。
