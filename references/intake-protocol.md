# 研究者基本情况摸底规范

> **加载时机**：Phase 0 最开始，在三问之前。
> **目的**：在追问"你在关心什么问题"之前，先了解研究者是谁、从哪来、到哪去，让追问有据可依。

## 一、为什么先摸底

空问"你在关心什么问题"容易得到模糊回答。如果先了解研究者的学科背景、历史沿革和研究基础，追问就能具体到：

- "你说之前做过平台劳动的收入研究，现在转向职业稳定性，是什么触发的？"
- "你导师的方向是劳动经济学，这个选题和导师方向的关系是？"
- "你提到有招聘数据，但没有骑手访谈，数据上够用吗？"

摸底不是审讯，是让后续每一步都基于研究者的真实处境。

## 二、六个摸底维度

### 1. 学科背景与研究方向

- 一级学科 / 二级学科
- 长期关注的研究方向（1-3 个关键词）
- 方法训练：计量 / 质性 / 混合 / 理论建模 / 实验设计
- 是否有跨学科背景

### 2. 当前阶段

- 本科 / 硕士 / 博士 / 青年教师 / 在职科研 / 其他
- 如果是学生：几年级、是否已开题、离毕业还有多久
- 如果是教师/科研人员：职称、是否需要完成考核指标

### 3. 历史沿革

- 之前做过什么研究（课题、论文、项目）
- 为什么转向现在这个方向（新政策、新数据、新现象、导师建议、个人兴趣）
- 有没有"半途而废"的方向及原因（避免重复踩坑）

### 4. 研究基础

- 已有数据：问卷、行政数据、爬虫数据、公开数据库、访谈记录
- 已有方法能力：是否掌握 Stata / R / Python / NVivo / 问卷设计 / 田野调查
- 合作方或导师支持：能否获取特定数据、是否有团队资源
- 已发表或已完成的成果

### 5. 交付目标

- 课程论文 / 学位论文 / 期刊论文 / 基金申报 / 研究构想 / 政策报告
- 如果是期刊：目标期刊层次和领域偏好
- 如果是基金：国家社科 / 自科 / 教育部 / 省级 / 横向

### 6. 时间约束

- 什么时候要交
- 是否有中间节点（开题、中期、预答辩）
- 能投入的实际时间（全职 / 兼职 / 周末）

## 三、执行方式

1. **一次性发问，不逐条审讯**：把六个维度用一段话或一个表格呈现给用户，请其一次性填写或口述。
2. **允许跳过**：用户可以标注"暂无"或"不确定"，但不能全部跳过。至少要提供学科背景、当前阶段和交付目标。
3. **记录到 `00_任务元信息.md`**：摸底结果作为 `01_三问与澄清.md` 的前置信息写入，协议中的 `researcher_profile` 字段记录结构化版本。
4. **追问必须引用摸底信息**：Phase 1 的三问不再是"你在关心什么问题"，而是基于摸底信息提出具体追问。

## 四、摸底结果如何影响后续流程

| 摸底维度 | 影响的后续阶段 |
|---|---|
| 学科背景与方法训练 | Phase 1 冻结协议的 `discipline_branch`；Phase 6.5 好问题卡的方法可行性判断 |
| 当前阶段与时间约束 | Phase 1 冻结协议的 `stakes`（课程论文=standard，学位论文/基金=high） |
| 历史沿革 | Phase 1 三问中的"为什么是现在"；Phase 2 扫描时避免重复已做过的方向 |
| 研究基础 | Phase 1.5 材料交互时引导用户提交已有数据；Phase 6.5 pilot 可行性判断 |
| 交付目标 | Phase 1 冻结协议的 `deliverable_type`；Phase 2 发表/申报窗口扫描的目标 |
| 时间约束 | Phase 6.5 pilot 的时间可行性；Phase 7 选题推荐的时间风险标注 |
