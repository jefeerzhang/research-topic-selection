# 用户材料交互与研判

> 加载时机：scope 闸门通过后、任何外部五维检索之前。

## 一、必须暂停交互

明确告诉用户：为了避免 Agent 用通用检索替代研究者自己的知识积累，需要先提供至少一项材料：

1. 与研究方向相关的文献；
2. 用户自己的文献笔记、阅读摘录或研究备忘；
3. 已有研究题目与摘要。

材料形式不设单一格式限制。明确接受：

- PDF；
- Word（`.doc` / `.docx`）；
- Markdown（`.md`）；
- 用户直接在对话框手输或粘贴的文字；
- 其他能够读取或通过现有工具提取内容的文件。

对话框中的文字应原样保存为 Markdown，标记 `origin=user-inline`；用户上传的文件标记
`origin=user-upload`。登记脚本接受任意非空文件，但 Agent 必须能够读取或提取其实际内容后才能完成研判。

此处是 BLOCKING 交互点。没有用户材料时不要自动检索、不要生成正式候选题、不要把 Agent 的背景知识伪装成用户基础。
如果用户没有文献或笔记，至少请其提供一个暂定题目和摘要；用户在对话中直接提供的文字必须原样保存为 Markdown 文件后登记。

## 二、登记材料

对每个文件运行：

```bash
python scripts/register_material.py --workdir <wd> --input <file> \
  --category literature --origin user-upload --label "用户说明"
```

类别：

- `literature`：论文、报告、政策研究等原始文献；
- `literature-note`：用户笔记、摘录、批注、研究备忘；
- `topic-abstract`：题目、摘要、研究构想或已有项目简介。

`origin` 必须是 `user-upload`（用户上传文件）或 `user-inline`（用户在对话中直接提供、原样保存为文件）。

脚本将文件复制到 `user_materials/` 并登记 SHA-256。不得登记空文件，也不得在登记后替换文件内容。

## 三、先读材料，再做研判

完整读取用户提供的材料。论文不得仅凭文件名、标题或首页猜测内容；无法读取的格式应先提取，仍失败则向用户说明并停在本阶段。

生成 `02_用户材料研判.md`，至少包含：

- **材料概览**：逐项列出材料 ID、类别、实际内容和证据边界；
- **用户已有研究问题**：从题目、摘要和笔记中提取，不替用户擅自换题；
- **用户已有证据与观点**：区分材料原文、用户判断与 Agent 推断；
- **材料之间的冲突**：不同文献、笔记或摘要是否存在理论、对象、方法或结论矛盾；
- **对后续检索的约束**：需要沿用的概念、对象、时间、方法、数据与排除范围；
- **尚待核验**：只有外部检索才能确认的事实、缺口和时效性判断。

每个已登记 `material_id` 必须在研判文件中出现。不要只写文件列表；研判需要说明材料如何改变后续问题界定和检索策略。

## 四、用户确认

向用户呈现简短研判摘要，并请其确认是否准确反映已有积累。用户纠正后更新 `02_用户材料研判.md`，再运行：

```bash
python scripts/selection_gate.py --workdir <wd> --enter materials
```

只有材料闸门 exit 0，才能进入五维扫描。用户材料负责提供研究者的起点、积累和约束，但不垄断后续证据。
Agent 可以检索并使用用户未提供的新文献、新数据、新政策和新解释，也可以据此修正或挑战用户的初步判断。
与用户材料直接相关的发现标记 `confirm/challenge/extend`；独立发现标记 `independent`。
