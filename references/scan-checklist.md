# 五维扫描执行清单（v1.5：用户材料约束下的外部扫描）

> **加载时机**：Phase 2 五维扫描时加载，且 materials 闸门必须已经 PASS。
> 原技能只说"if current information is needed, use appropriate search/research tools"，
> 本清单把工具、深度、反确认偏差**写死**，避免扫描质量全靠 Agent 自觉。

## 〇、扫描总原则

- **用户材料优先**：先读取 `02_用户材料研判.md` 和 `review/user_material_manifest.json`，把“尚待核验”转成检索问题，把“对后续检索的约束”转成纳排边界。
- **来源关系标注**：回应用户材料的发现说明它是确认、挑战还是扩展哪个材料 ID；Agent 独立发现的新证据标记 `independent`。新证据可以修正原题，但不得无说明地替换用户研究对象。
- **检索先行**：凡涉时效性（政策、近期文献、数据窗口、申报指南），必须实际调用检索，禁止凭记忆编造。
- **反确认偏差**：每维至少主动寻找 1 条**反面/竞争性**证据（不支持你预设方向的文献、失败的政策试点、不可得的数据）。
- **来源留痕**：每条关键结论附来源（URL / 文献 DOI / 政策文号 / 数据集名），写入 `03_五维扫描.md` 和 `review/evidence_registry.jsonl`。
- **深度阈值**：每维至少 3 条有来源的实质发现；薄弱维度明确写"暂无可靠来源，风险标注"。

## 一、政策扫描

工具：web/google 检索、政府官网站内（gov.cn / 部委 / 地方）、政策数据库。
检索式示例：
```
"[主题]" site:gov.cn 政策/通知/指导意见
"[主题]" 十四五/十五五 规划
"[主题]" 试点 示范区
```
关注：国家级方向、地方试点、治理任务、监管变化、政策窗口期（何时申报/结题）。

## 二、学术文献扫描

工具（优先级）：
1. 本地 anysearch / web_google 等联网检索。
2. 学术 API 检索（OpenAlex/Semantic Scholar/arXiv/PubMed 等，若环境可用）。
3. 中文：CNKI/万方 site 检索 + 核心期刊清单（见 discipline-branches.md）。
关注：主要学术分支、经典问题、近 3 年趋势、核心争论、前沿方向。
**禁止编造文献**：未实际检索到的"文献"一律不写，或明确标注"[待检索核实]"。

## 三、现实实践扫描

工具：web 检索、新闻/行业报告、案例库。
关注：地方实践、典型案例、新兴现象、组织模式、田野变化。
每维至少 1 条具体案例（地名/机构名/时间），避免"许多地区已开始……"式空话。

## 四、数据/材料扫描

工具：开放数据平台检索（国家数据/统计年鉴/世界银行/公开调查）、用户材料盘点。
关注：公开数据集、地方行政数据可得性、调查问卷可行性、政策文本、访谈/案例可达性。
**可行性分级**：可得 / 需申请 / 不可得（标红，影响选题可执性）。

## 五、发表/申报窗口扫描

工具：期刊官网、基金委/社科规划办指南、会议征稿。
关注：目标期刊主题/特刊、基金申报指南（国家社科/自科/教育部）、委托课题、政策报告窗口。
标注窗口期与截止时间（若有）。

## 六、确认偏差自检（进入 scan 审查前的硬性动作）

在 `03_五维扫描.md` 末尾追加：
```markdown
## 反确认偏差记录
- 找到的反面证据：
- 未找到但预期存在的反面证据（说明可能原因）：
- 扫描覆盖盲区：
```
该段缺失或全空，scan 审查节点应判 P0。
